= Spring Boot Plugins
:toc: left
:toclevels: 3

The Spring Boot plugins provide convention-based configuration for Spring Boot applications and libraries, including dependency management, test dependencies, and DevTools integration.

== Spring Boot Application Plugin

Configures a project as a Spring Boot executable application with `bootJar`, `bootRun`, and DevTools support.

Plugin ID:: `dev.daymor.ultimanexus.jvm.gradle.feature.spring-boot-application`

[source,kotlin]
----
plugins {
    alias(libs.plugins.ultimanexus.jvm.feature.spring.boot.application)
}
----

=== What It Does

* Applies the `org.springframework.boot` Gradle plugin
* Enables `bootJar` and `bootRun` tasks
* Creates an executable fat JAR containing all dependencies
* Configures `productionRuntimeClasspath` and `developmentOnly` configurations
* Adds Spring Boot DevTools as a `developmentOnly` dependency

=== Running

[source,bash]
----
./gradlew bootRun    # Run the application
./gradlew bootJar    # Build executable fat JAR
----

== Spring Boot Library Plugin

Configures a project as a Spring Boot library with dependency management but without the full Spring Boot plugin.
Produces a standard JAR (not executable).

Plugin ID:: `dev.daymor.ultimanexus.jvm.gradle.feature.spring-boot-library`

[source,kotlin]
----
plugins {
    alias(libs.plugins.ultimanexus.jvm.feature.spring.boot.library)
}
----

=== What It Does

* Applies Spring Dependency Management (BOM import without `bootJar`/`bootRun`)
* Applies the `java-library` plugin for `api`/`implementation` separation
* Produces a standard JAR suitable for publishing to Maven Central

=== Application vs. Library

[%autowidth.stretch,cols="1,1,1,1"]
|===
|Feature |Application |Library |Notes

|`bootJar` task
|Yes
|No
|Only applications produce executable JARs

|`bootRun` task
|Yes
|No
|Only applications can be run directly

|`jar` task
|Disabled
|Enabled
|Library mode produces standard JARs

|DevTools
|Yes
|No
|DevTools is only for development applications

|Publishing
|No
|Yes
|Libraries are published to Maven Central

|Dependency management
|Spring Boot BOM
|Spring Boot BOM
|Both get version-managed dependencies
|===

== Spring Boot Test Dependencies Plugin

Adds common Spring Boot test dependencies.

Plugin ID:: `dev.daymor.ultimanexus.jvm.gradle.dependency.spring-boot-test`

[source,kotlin]
----
plugins {
    alias(libs.plugins.ultimanexus.jvm.dependency.spring.boot.test)
}
----

=== What It Provides

* `spring-boot-starter-test` as `testImplementation`
* JSR-305 annotations as `testCompileOnly`

== Spring Boot Bundles

=== Spring Boot Complete Application

Full-featured Spring Boot application with quality checks, testing, and reporting.

Plugin ID:: `dev.daymor.ultimanexus.jvm.gradle.bundle.spring-boot-complete-application`

[source,kotlin]
----
plugins {
    alias(libs.plugins.ultimanexus.jvm.bundle.spring.boot.complete.application)
}
----

Includes:

* Gradle project base (lifecycle, identity, dependency-rules, repositories)
* Quality checks (Checkstyle, PMD, SpotBugs, Spotless)
* Testing (unit, integration, functional, performance)
* Reporting (code coverage, SBOM, test reports)
* Java compilation and Javadoc
* Spring Boot application (`bootJar`, `bootRun`, DevTools)
* Spring Boot test dependencies
* JSpecify nullability annotations

=== Spring Boot Complete Library

Full-featured Spring Boot library with publishing.

Plugin ID:: `dev.daymor.ultimanexus.jvm.gradle.bundle.spring-boot-complete-library`

[source,kotlin]
----
plugins {
    alias(libs.plugins.ultimanexus.jvm.bundle.spring.boot.complete.library)
}
----

Includes everything in the application bundle except `bootJar`/`bootRun`/DevTools, plus:

* `java-library` plugin (API/implementation separation)
* Maven Central publishing with sources and Javadoc JARs

== Spring Boot AOT Plugin

Configures Spring Boot Ahead-of-Time (AOT) compilation and excludes AOT-generated code from quality checks.

Plugin ID:: `dev.daymor.ultimanexus.jvm.gradle.feature.spring-boot-aot`

[source,kotlin]
----
plugins {
    alias(libs.plugins.ultimanexus.jvm.feature.spring.boot.aot)
}
----

=== What It Does

* Enables Spring Boot AOT processing (`processAot`, `processTestAot` tasks)
* Suppresses `-Xlint:unchecked` and `-Xlint:rawtypes` warnings on AOT compilation tasks
* Disables Checkstyle, PMD, and SpotBugs tasks for AOT source sets
* Disables AOT processing when no main source exists (e.g., starter modules)

=== Configuration

[source,properties]
----
springBootAot.suppressWarnings=true
----

== Spring Boot Version Management

The Spring Boot version is managed through your version catalog.
Add the following to `gradle/libs.versions.toml`:

[source,toml]
----
[versions]
spring-boot = "4.0.1"

[plugins]
spring-boot = { id = "org.springframework.boot", version.ref = "spring-boot" }

[libraries]
spring-boot-devtools = { module = "org.springframework.boot:spring-boot-devtools", version.ref = "spring-boot" }
spring-boot-starter-test = { module = "org.springframework.boot:spring-boot-starter-test", version.ref = "spring-boot" }
----

== Common Tasks

[source,bash]
----
# Run Spring Boot application
./gradlew bootRun

# Build executable JAR
./gradlew bootJar

# Build standard JAR (library mode)
./gradlew jar

# Run quality checks
./gradlew qualityCheck

# Run all tests
./gradlew allTest
----
