= Test Plugins
:toc: left
:toclevels: 3

The test plugins provide comprehensive testing infrastructure with support for multiple test types, configurable test suites, code coverage, and shared test fixtures.

== Test Bundle

The recommended way to apply all testing infrastructure at once.

Plugin ID:: `dev.daymor.ultimanexus.jvm.gradle.bundle.test`

[source,kotlin]
----
plugins {
    alias(libs.plugins.ultimanexus.jvm.bundle.test)
}
----

This bundle includes:

* Unit test configuration (JUnit 5, JaCoCo, ByteBuddy agent)
* Configurable test suites (integration, functional, performance)
* `allTest` task to run all test types

=== Running Tests

[source,bash]
----
# Unit tests only
./gradlew test

# Individual test suites
./gradlew integrationTest
./gradlew functionalTest
./gradlew performanceTest

# ALL test types in one command
./gradlew allTest
----

== Unit Test Plugin

Configures the default `test` task with JUnit Jupiter, JaCoCo coverage, and ByteBuddy agent for mocking final classes and static methods.

Plugin ID:: `dev.daymor.ultimanexus.jvm.gradle.test.test`

[source,kotlin]
----
plugins {
    alias(libs.plugins.ultimanexus.jvm.test.test)
}
----

=== What It Provides

* *JUnit 5 (Jupiter)* -- test framework
* *JaCoCo* -- code coverage measurement
* *ByteBuddy agent* -- enables mocking of final classes and static methods (e.g., with Mockito)
* *SLF4J Simple* -- test logging output
* *Parallel execution* -- configurable parallel fork count

=== Extension Configuration

[source,kotlin]
----
testConfig {
    maxHeapSize.set("2g")
    maxParallelForks.set(4)
    showStandardStreams.set(false)
    fileEncoding.set("UTF-8")
    useByteBuddyAgent.set(true)
}
----

=== Properties Configuration

[source,properties]
----
test.maxHeapSize=2g
test.maxParallelForks=4
test.showStandardStreams=false
test.fileEncoding=UTF-8
test.useByteBuddyAgent=true
----

== Test Suites Plugin

Creates configurable test suites beyond the default unit tests.
Each suite gets its own source set, task, and dedicated configuration.

Plugin ID:: `dev.daymor.ultimanexus.jvm.gradle.test.test-suites`

[source,kotlin]
----
plugins {
    alias(libs.plugins.ultimanexus.jvm.test.test.suites)
}
----

=== Default Suites

By default, three test suites are created:

[cols="1,2,2"]
|===
|Suite |Source Directory |Task Name

|Integration tests
|`src/integrationTest/java`
|`integrationTest`

|Functional tests
|`src/functionalTest/java`
|`functionalTest`

|Performance tests
|`src/performanceTest/java`
|`performanceTest`
|===

=== Customizing Suites

Override the default suites or add custom ones:

[source,kotlin]
----
testSuites {
    // Replace default suites with a custom list
    suites.set(listOf("integrationTest", "functionalTest", "smokeTest", "e2eTest"))

    // Global defaults (inherited by all suites)
    maxHeapSize.set("2g")
    maxParallelForks.set(4)
    showStandardStreams.set(true)
    fileEncoding.set("UTF-8")
    useByteBuddyAgent.set(true)
    useJacoco.set(true)

    // Per-suite overrides
    suiteConfig("performanceTest") {
        useByteBuddyAgent.set(false)  // Disable mocking overhead for benchmarks
        useJacoco.set(false)           // Disable coverage for JMH benchmarks
    }
    suiteConfig("smokeTest") {
        maxParallelForks.set(1)        // Run smoke tests sequentially
        maxHeapSize.set("512m")
    }
}
----

Or via `gradle.properties`:

[source,properties]
----
# Override default suites
test.suites=integrationTest,functionalTest,smokeTest,e2eTest

# Global defaults
test.maxHeapSize=2g
test.maxParallelForks=4
test.showStandardStreams=true
test.fileEncoding=UTF-8
test.useByteBuddyAgent=true
test.useJacoco=true

# Per-suite overrides
test.suite.performanceTest.useByteBuddyAgent=false
test.suite.performanceTest.useJacoco=false
test.suite.smokeTest.maxParallelForks=1
test.suite.smokeTest.maxHeapSize=512m
----

=== ByteBuddy Agent

The ByteBuddy agent is enabled by default for all suites except performance tests.
It allows mocking frameworks (Mockito, etc.) to mock final classes and static methods.

Performance tests disable ByteBuddy by default to avoid instrumentation overhead during benchmarking.

=== JaCoCo Coverage

JaCoCo code coverage is enabled by default for all suites except performance tests.
Performance tests disable JaCoCo because JMH forks its own JVM processes, making JaCoCo instrumentation ineffective for benchmark code.

== Test Fixtures Plugin

Enables sharing test utilities across projects using Gradle's `java-test-fixtures` feature.

Plugin ID:: `dev.daymor.ultimanexus.jvm.gradle.test.test-fixtures`

[source,kotlin]
----
plugins {
    alias(libs.plugins.ultimanexus.jvm.test.test.fixtures)
}
----

=== Source Location

[source]
----
src/testFixtures/java/     # Shared test utilities
----

=== Producing Fixtures

Place reusable test helpers, builders, and base classes in `src/testFixtures/java/`:

[source,java]
.src/testFixtures/java/com/example/util/CompanyTestUtil.java
----
public class CompanyTestUtil {

    public static Company createValidCompany() {
        return Company.builder()
            .name("Test Company")
            .build();
    }
}
----

=== Consuming Fixtures

In other projects that depend on this module:

[source,kotlin]
----
dependencies {
    testImplementation(testFixtures(project(":core")))

    // Also available for other test suites
    integrationTestImplementation(testFixtures(project(":core")))
}
----

== Directory Structure

With the test bundle applied, the source directory layout is:

[source]
----
src/
+-- main/java/              # Production source code
+-- test/java/              # Unit tests
+-- integrationTest/java/   # Integration tests
+-- functionalTest/java/    # Functional tests
+-- performanceTest/java/   # Performance tests (JMH benchmarks)
+-- testFixtures/java/      # Shared test utilities
----

Each test source set inherits the dependencies of `test` (JUnit 5, assertion libraries, etc.) and has access to `main` output classes.

== Integration with Reports

The test plugins work seamlessly with the xref:report-plugins.adoc[report plugins]:

* *Code coverage:* Each test suite gets its own JaCoCo report task (e.g., `testCodeCoverageReport`, `integrationTestCodeCoverageReport`), plus a combined `allTestCodeCoverageReport`.
* *Test reports:* Each suite gets an aggregated test report task (e.g., `testAggregateTestReport`), plus a combined `allTestAggregateTestReport`.

See the xref:report-plugins.adoc[Report Plugins] section for details.

== Properties Reference

|===
|Property |Default |Plugin |Description

|`test.maxHeapSize`
|`1g`
|`test`, `test-suites`
|Maximum heap size for test JVMs

|`test.maxParallelForks`
|_(half of CPUs)_
|`test`, `test-suites`
|Number of parallel test processes

|`test.showStandardStreams`
|`true`
|`test`, `test-suites`
|Show stdout/stderr in test output

|`test.fileEncoding`
|`UTF-8`
|`test`, `test-suites`
|File encoding for test processes

|`test.useByteBuddyAgent`
|`true`
|`test`, `test-suites`
|Enable ByteBuddy agent for mocking

|`test.useJacoco`
|`true`
|`test-suites`
|Enable JaCoCo code coverage for suites

|`test.suites`
|`integrationTest,functionalTest,performanceTest`
|`test-suites`
|Comma-separated list of test suites to create

|`test.suite.<name>.maxHeapSize`
|_(inherits global)_
|`test-suites`
|Per-suite maximum heap size

|`test.suite.<name>.maxParallelForks`
|_(inherits global)_
|`test-suites`
|Per-suite parallel test processes

|`test.suite.<name>.showStandardStreams`
|_(inherits global)_
|`test-suites`
|Per-suite stdout/stderr setting

|`test.suite.<name>.fileEncoding`
|_(inherits global)_
|`test-suites`
|Per-suite file encoding

|`test.suite.<name>.useByteBuddyAgent`
|`true` (except `performanceTest`: `false`)
|`test-suites`
|Per-suite ByteBuddy agent setting

|`test.suite.<name>.useJacoco`
|`true` (except `performanceTest`: `false`)
|`test-suites`
|Per-suite JaCoCo coverage setting
|===
