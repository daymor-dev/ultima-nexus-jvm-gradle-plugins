= Java Plugins
:description: Java compilation, Javadoc generation, application/library configuration, and nullability annotations.
:toc: left
:toclevels: 3

Java plugins configure compilation settings, Javadoc generation, application and library project types, and JSpecify nullability annotations.

TIP: For Lombok annotation processing, see xref:lombok-plugins.adoc[Lombok Plugins].

== Compile Java Plugin

Configures Java compilation settings including JDK toolchain version and compiler options.

Plugin ID:: `dev.daymor.ultimanexus.jvm.gradle.feature.compile-java`

=== Extension Configuration

[source,kotlin]
----
compileJavaConfig {
    jdkVersion.set(21)
}
----

=== Properties Configuration

[source,properties]
----
jdkVersion=21
----

=== Default Settings

[cols="1,2"]
|===
|Setting |Value

|JDK version
|25

|Fork compilation
|Enabled

|Encoding
|UTF-8

|Compiler flags
|`-parameters`, `-implicit:none`, `-Werror`, `-Xlint:all,-serial,-processing`
|===

=== Reproducible Builds

Archives produced by this plugin are configured for reproducibility:

* Consistent file order
* No timestamps preserved
* Standard file permissions (0664 for files, 0775 for directories)

== Java Application Plugin

Configures a project as a runnable Java application with `run`, `distZip`, and `distTar` tasks.

Plugin ID:: `dev.daymor.ultimanexus.jvm.gradle.feature.java-application`

[source,kotlin]
----
plugins {
    alias(libs.plugins.ultimanexus.jvm.feature.java.application)
}
----

=== What It Does

* Applies the Gradle `application` plugin
* Provides the `run` task for executing the application
* Provides distribution tasks (`distZip`, `distTar`) for packaging

== Java Library Plugin

Configures a project as a reusable Java library with `api` and `implementation` dependency separation.

Plugin ID:: `dev.daymor.ultimanexus.jvm.gradle.feature.java-library`

[source,kotlin]
----
plugins {
    alias(libs.plugins.ultimanexus.jvm.feature.java.library)
}
----

=== What It Does

* Applies the Gradle `java-library` plugin
* Enables `api` and `implementation` dependency configurations for proper transitive dependency management

== JSpecify Plugins

Adds https://jspecify.dev/[JSpecify] nullability annotations.

=== JSpecify Bundle (Recommended)

Adds JSpecify to both main and test sources.

Plugin ID:: `dev.daymor.ultimanexus.jvm.gradle.bundle.jspecify`

[source,kotlin]
----
plugins {
    alias(libs.plugins.ultimanexus.jvm.bundle.jspecify)
}
----

=== JSpecify (Main Only)

Plugin ID:: `dev.daymor.ultimanexus.jvm.gradle.dependency.jspecify`

Adds JSpecify as `compileOnly` for main sources.

=== JSpecify Test (Test Only)

Plugin ID:: `dev.daymor.ultimanexus.jvm.gradle.dependency.jspecify-test`

Adds JSpecify as `testCompileOnly` for test sources.

=== Usage

[source,java]
----
import org.jspecify.annotations.NullMarked;
import org.jspecify.annotations.Nullable;

@NullMarked
public class MyService {

    public void process(@Nullable String input) {
        // ...
    }
}
----
