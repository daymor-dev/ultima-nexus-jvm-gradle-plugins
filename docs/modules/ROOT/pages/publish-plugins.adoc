= Publishing Plugins
:toc: left
:toclevels: 3

The publishing plugins configure Maven Central publishing with POM metadata, GPG signing, sources JAR, and Javadoc JAR generation.

== Publishing Approaches

There are two approaches to publishing to Maven Central:

=== Approach 1: Settings Plugin (Recommended)

The settings plugin automatically aggregates all publishable subprojects.
No manual module listing required.

[source,kotlin]
.settings.gradle.kts
----
plugins {
    id("dev.daymor.ultimanexus.jvm.gradle.feature.publish-maven-central") version "latest.release"
}
----

[source,kotlin]
.Subproject build.gradle.kts
----
plugins {
    alias(libs.plugins.ultimanexus.jvm.feature.publish.java)
}
----

[source,bash]
----
./gradlew publishAggregationToCentralPortal
----

TIP: If you use the `ultima-nexus-jvm-settings` or `ultima-nexus-jvm-complete` bundle, the Maven Central settings plugin is automatically applied when any subproject uses a publish plugin.

=== Approach 2: Build Plugin (Explicit Control)

For projects that need explicit control over which modules to publish.

[source,kotlin]
.Root build.gradle.kts
----
plugins {
    alias(libs.plugins.ultimanexus.jvm.feature.publish.maven.central.aggregation)
}

dependencies {
    nmcpAggregation(project(":core"))
    nmcpAggregation(project(":api"))
    nmcpAggregation(project(":web"))
}
----

Or via `gradle.properties`:

[source,properties]
----
publishModules=:core,:api,:web
----

== Publish Plugin

Configures Maven publishing with POM metadata and GPG signing.

Plugin ID:: `dev.daymor.ultimanexus.jvm.gradle.feature.publish`

This is the base publish plugin.
It configures:

* Maven POM metadata (project URL, license, developer info, SCM)
* GPG signing of all publications
* Maven Central-compatible publication

=== Extension Configuration

[source,kotlin]
----
publishConfig {
    // Project information
    projectUrl.set("https://github.com/example/project")
    inceptionYear.set("2025")

    // License
    licenseName.set("The Apache License, Version 2.0")
    licenseUrl.set("http://www.apache.org/licenses/LICENSE-2.0.txt")

    // Developer information
    developerId.set("developer")
    developerName.set("Developer Name")
    developerEmail.set("developer@example.com")
    developerUrl.set("https://example.com")
    developerOrganization.set("Organization")
    developerOrganizationUrl.set("https://organization.com")

    // SCM information
    scmUrl.set("https://github.com/example/project")
    scmConnection.set("scm:git:git://github.com/example/project.git")
    scmDeveloperConnection.set("scm:git:ssh://github.com/example/project.git")
}
----

=== Properties Configuration

[source,properties]
----
# Project information
publish.projectUrl=https://github.com/example/project
publish.inceptionYear=2025

# License
publish.licenseName=The Apache License, Version 2.0
publish.licenseUrl=http://www.apache.org/licenses/LICENSE-2.0.txt

# Developer information
publish.developerId=developer
publish.developerName=Developer Name
publish.developerEmail=developer@example.com
publish.developerUrl=https://example.com
publish.developerOrganization=Organization
publish.developerOrganizationUrl=https://organization.com

# SCM information
publish.scmUrl=https://github.com/example/project
publish.scmConnection=scm:git:git://github.com/example/project.git
publish.scmDeveloperConnection=scm:git:ssh://github.com/example/project.git
----

== Signing Configuration

GPG signing is required for Maven Central.
Configure signing credentials via `gradle.properties` or environment variables.

=== Using gradle.properties

[source,properties]
.~/.gradle/gradle.properties (user-level, NOT project-level)
----
signingKey=-----BEGIN PGP PRIVATE KEY BLOCK-----\n...
signingPassword=your-key-password
----

IMPORTANT: Never commit signing keys to version control.
Store them in `~/.gradle/gradle.properties` (user home directory) or use environment variables.

=== Using Environment Variables

[source,bash]
----
export SIGNINGKEY="-----BEGIN PGP PRIVATE KEY BLOCK-----..."
export SIGNINGPASSWORD="your-key-password"
----

Environment variables take precedence over `gradle.properties` values.

== Java Publishing Plugin

Extends the base publish plugin with sources JAR and Javadoc JAR generation.

Plugin ID:: `dev.daymor.ultimanexus.jvm.gradle.feature.publish-java`

[source,kotlin]
----
plugins {
    alias(libs.plugins.ultimanexus.jvm.feature.publish.java)
}
----

=== What It Does

This plugin automatically:

* Applies the base publish plugin (POM metadata + signing)
* Generates a sources JAR (`sourcesJar` task)
* Generates a Javadoc JAR (`javadocJar` task)
* Attaches both JARs to the Maven publication

No additional configuration is required beyond the base publish plugin settings.

== Maven Central Aggregation Plugin

For multi-module projects, aggregates all publications and uploads them to Maven Central in a single operation.

Plugin ID:: `dev.daymor.ultimanexus.jvm.gradle.feature.publish-maven-central-aggregation`

Apply in root `build.gradle.kts`:

[source,kotlin]
----
plugins {
    alias(libs.plugins.ultimanexus.jvm.feature.publish.maven.central.aggregation)
}
----

=== Specifying Modules

Option 1 -- In the build file:

[source,kotlin]
----
dependencies {
    nmcpAggregation(project(":core"))
    nmcpAggregation(project(":api"))
    nmcpAggregation(project(":web"))
}
----

Option 2 -- In `gradle.properties`:

[source,properties]
----
publishModules=:core,:api,:web
----

=== Maven Central Credentials

[source,properties]
.gradle.properties (or environment variables)
----
mavenCentralUsername=<your-token-username>
mavenCentralPassword=<your-token-password>
----

Or:

[source,bash]
----
export MAVENCENTRALUSERNAME=<your-token-username>
export MAVENCENTRALPASSWORD=<your-token-password>
----

=== Publishing Command

[source,bash]
----
./gradlew publishAggregationToCentralPortal
----

== Gradle Plugin Publishing

For publishing Gradle plugins to the Gradle Plugin Portal.

Plugin ID:: `dev.daymor.ultimanexus.jvm.gradle.feature.gradle-plugin-publish`

This plugin is used internally by the Ultima Nexus plugin project itself.
It configures plugin metadata for the Gradle Plugin Portal.

=== Configuration

[source,properties]
----
groupId=dev.daymor.ultimanexus.jvm.gradle
version=latest.release
pluginWebsite=https://github.com/your/repo
pluginVcsUrl=https://github.com/your/repo
pluginTags=java,convention,quality

# Per-plugin metadata
plugin.check.checkstyle.displayName=Ultima Nexus - Checkstyle
plugin.check.checkstyle.description=Configures Checkstyle code quality checks
----

=== Publishing

[source,bash]
----
export GRADLE_PUBLISH_KEY=your-key
export GRADLE_PUBLISH_SECRET=your-secret
./gradlew publishPlugins
----

== Publishing Workflow

A complete end-to-end workflow for publishing to Maven Central:

. Configure `groupId` and `version` in `gradle.properties`.
. Configure publishing metadata (extension DSL or `gradle.properties`).
. Set up GPG signing credentials (environment variables or `~/.gradle/gradle.properties`).
. Set up Maven Central credentials (environment variables or `~/.gradle/gradle.properties`).
. Build and verify: `./gradlew build`.
. Publish: `./gradlew publishAggregationToCentralPortal`.

== Local Publishing

For testing publications locally before pushing to Maven Central:

[source,bash]
----
./gradlew publishToMavenLocal
----

This publishes artifacts to `~/.m2/repository/` where they can be consumed by other local projects.
