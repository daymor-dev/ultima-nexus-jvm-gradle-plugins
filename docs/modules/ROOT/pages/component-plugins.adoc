= Component Plugins
:toc: left
:toclevels: 3

Component plugins define the type and role of a Gradle project.
They determine whether a project is an application, a library, a starter, or a version platform, and they aggregate the appropriate feature plugins for that role.

== Choosing a Component Type

[cols="1,2,2"]
|===
|Component |Description |Key Characteristics

|*Application*
|Executable program (Spring Boot, CLI tool, service)
|Produces a runnable artifact (`bootJar`, `run` task); not published to Maven Central

|*Library*
|Reusable code consumed by other projects
|Produces a JAR with sources and Javadoc; published to Maven Central; `api` / `implementation` dependency separation

|*Starter*
|Dependency aggregator with no source code
|No `src/` directory; aggregates `api` dependencies for consumers to pull in transitively

|*Platform*
|Version constraint manager (BOM)
|Uses `java-platform`; defines dependency version constraints for other projects to consume

|*Root*
|Root project in a multi-module build
|Aggregates reports, configures root lifecycle tasks; no production source code
|===

== Application Bundles

=== Ultima Nexus JVM Application (Full-Featured)

The most comprehensive application bundle, including Spring Boot, GraalVM Native Image, Lombok, quality checks, and documentation.

Plugin ID:: `dev.daymor.ultimanexus.jvm.gradle.bundle.ultima-nexus-jvm-application`

[source,kotlin]
----

    alias(libs.plugins.ultimanexus.jvm.bundle.ultima.nexus.jvm.application)
}
----

Includes: Spring Boot application + GraalVM Native Image + AOT + quality checks + testing + reporting + Lombok + JSpecify + documentation.

== Language-Agnostic Bundle

=== JVM Complete

For Kotlin, Scala, Groovy, or other JVM languages.
Provides quality checks, testing, and reporting without Java-specific features.

Plugin ID:: `dev.daymor.ultimanexus.jvm.gradle.bundle.jvm-complete`

[source,kotlin]
----
plugins {
    alias(libs.plugins.ultimanexus.jvm.bundle.jvm.complete)
    kotlin("jvm")  // Add your language plugin
}
----

Includes: Gradle project base + quality checks + testing + reporting.

Does NOT include: `compile-java`, `javadoc`, `jspecify`, `lombok`, or any Spring Boot plugin.

== Platform Bundle

For BOM (Bill of Materials) projects that manage dependency versions.

Plugin ID:: `dev.daymor.ultimanexus.jvm.gradle.bundle.platform`

[source,kotlin]
----
plugins {
    alias(libs.plugins.ultimanexus.jvm.bundle.platform)
}
----

Includes: `version-platform` + `identity` + `dependency-versions`.

IMPORTANT: The platform bundle applies `java-platform` and cannot be combined with `java` or `java-library` plugins.

[source,kotlin]
.Example platform build.gradle.kts
----
plugins {
    alias(libs.plugins.ultimanexus.jvm.bundle.platform)
}

// Declare version constraints for your project
dependencies {
    constraints {
        api(libs.guava)
        api(libs.jackson.core)
    }
}
----

== Starter Bundle

For starter subprojects that aggregate dependencies without containing source code.

Plugin ID:: `dev.daymor.ultimanexus.jvm.gradle.bundle.starter`

[source,kotlin]
----
plugins {
    alias(libs.plugins.ultimanexus.jvm.bundle.starter)
}

dependencies {
    api(libs.spring.boot.starter.web)
    api(libs.spring.boot.starter.validation)
    api(project(":my-common-utils"))
}
----

Includes: `java-library` + lifecycle + identity + dependency-rules + publish.

== Root Project Bundle

plugins {

}
----

Includes: `lifecycle.root` + `dependencies.root` + `format-gradle.root` + `aggregation`.
