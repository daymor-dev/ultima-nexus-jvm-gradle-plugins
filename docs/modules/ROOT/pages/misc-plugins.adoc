= Miscellaneous Plugins
:description: Aggregation, version platforms, starters, root projects, and top-level bundles.
:toc: left
:toclevels: 3

The `plugins-misc` module provides utility plugins for project structure roles (root, starter, platform) and top-level bundles that compose plugins from all other modules.

== Aggregation Plugin

Aggregates reports (JaCoCo, test reports, SBOM, plugin analysis) from subprojects into the root project.

Plugin ID:: `dev.daymor.ultimanexus.jvm.gradle.feature.aggregation`

[source,kotlin]
----
plugins {
    alias(libs.plugins.ultimanexus.jvm.feature.aggregation)
}
----

Conditionally applies aggregation when the corresponding report plugins are present in subprojects.

== Version Platform Plugin

Creates a Java Platform (BOM) with version constraints from the version catalog.

Plugin ID:: `dev.daymor.ultimanexus.jvm.gradle.feature.version-platform`

[source,kotlin]
----
plugins {
    alias(libs.plugins.ultimanexus.jvm.feature.version.platform)
}
----

=== Configuration

[source,kotlin]
----
versionPlatform {
    includeSpringBootBom = true
    springBootBomLibrary = "spring-boot-dependencies"
}
----

[source,properties]
----
versionPlatform.includeSpringBootBom=false
versionPlatform.springBootBomLibrary=spring-boot-dependencies
----

When the `org.springframework.boot` plugin is applied, the Spring Boot BOM is automatically included.

IMPORTANT: This plugin applies `java-platform` and cannot be combined with `java` or `java-library`.

== Use All Catalog Versions Plugin

Applies `java-platform` and creates strict version constraints for every entry in the version catalog.

Plugin ID:: `dev.daymor.ultimanexus.jvm.gradle.feature.use-all-catalog-versions`

[source,kotlin]
----
plugins {
    alias(libs.plugins.ultimanexus.jvm.feature.use.all.catalog.versions)
}
----

IMPORTANT: Use only in dedicated platform/BOM projects. Cannot be combined with `java` or `java-library`.

== Starter Bundle

For starter subprojects that aggregate dependencies without containing source code.

Plugin ID:: `dev.daymor.ultimanexus.jvm.gradle.bundle.starter`

[source,kotlin]
----
plugins {
    alias(libs.plugins.ultimanexus.jvm.bundle.starter)
}

dependencies {
    api(libs.spring.boot.starter.web)
    api(project(":my-common-utils"))
}
----

Includes: `java-library` + lifecycle + identity + dependency-rules + publish.

== Platform Bundle

For BOM (Bill of Materials) projects that manage dependency versions.

Plugin ID:: `dev.daymor.ultimanexus.jvm.gradle.bundle.platform`

[source,kotlin]
----

    alias(libs.plugins.ultimanexus.jvm.bundle.platform)
}

    constraints {
        api(libs.guava)
        api(libs.jackson.core)
    }
}
----

Includes: `version-platform` + `identity` + `dependency-versions`.

== JVM Complete Bundle

For Kotlin, Scala, Groovy, or other JVM languages. Provides quality checks, testing, and reporting without Java-specific features.

Plugin ID:: `dev.daymor.ultimanexus.jvm.gradle.bundle.jvm-complete`

[source,kotlin]
----
plugins {
    alias(libs.plugins.ultimanexus.jvm.bundle.jvm.complete)
    kotlin("jvm")  // Add your language plugin
}
----

Includes: Gradle project base + quality checks + testing + reporting.

Does NOT include: `compile-java`, `javadoc`, `jspecify`, `lombok`, or any Spring Boot plugin.

== Ultima Nexus JVM Application Bundle

The all-in-one bundle for Ultima Nexus framework applications. **No dependency configuration needed** ã the plugin automatically adds the starter, annotation module, and annotation processor.

Plugin ID:: `dev.daymor.ultimanexus.jvm.gradle.bundle.ultima-nexus-jvm-application`

[source,kotlin]
----
plugins {
    alias(libs.plugins.ultimanexus.jvm.bundle.ultima.nexus.jvm.application)
}
// That's it ã no dependencies block needed
----

Includes:

* Spring Boot application with GraalVM Native Image and AOT support
* Quality checks (Checkstyle, PMD, SpotBugs, Spotless)
* Testing infrastructure (unit, integration, functional, performance)
* Lombok and JSpecify annotation processing
* Documentation (auto-detected via `antora-playbook.yml`)
* **Ultima Nexus starter** (`implementation` ã predefined or plain, configurable)
* **Ultima Nexus annotations** (`compileOnly` ã `@GenerateCrud`, `@GenerateApplication`, etc.)
* **Ultima Nexus processor** (`annotationProcessor` ã selected based on application type and Hibernate setting)

=== Configuration

[cols="2,1,3"]
|===
| Property | Default | Description

| `ultimaNexusJvm.applicationType`
| `spring-rest-api`
| Application type. Determines which starter and processor to use. Supported: `spring-rest-api`, `entity-only`.

| `ultimaNexusJvm.usePredefinedStarter`
| `true`
| Use the opinionated starter. Set to `false` for the minimal starter. Ignored for `entity-only`.

| `ultimaNexusJvm.useHibernate`
| `true`
| Use Hibernate generation. Set to `false` for plain JPA identity methods.
|===

=== Processor Selection

The annotation processor is selected automatically based on the application type and Hibernate setting:

[cols="1,1,2"]
|===
| Application Type | Hibernate | Processor

| `spring-rest-api`
| `true` (default)
| `processor-java-spring-hibernate` ã full hexagonal stack with Hibernate

| `spring-rest-api`
| `false`
| `processor-java-spring-application` ã full hexagonal stack with plain identity

| `entity-only`
| `true`
| `processor-java-hibernate` ã entity generation only with Hibernate

| `entity-only`
| `false`
| `processor-java` ã entity generation only with plain identity
|===

=== Version Catalog Overrides

To override the auto-included dependencies, define these entries in your `libs.versions.toml`:

[source,toml]
----
[libraries]
ultima-nexus-jvm-starter = { module = "dev.daymor.ultima-nexus.jvm:starter-predefined-spring-rest-api-application", version = "latest.release" }
ultima-nexus-jvm-annotation = { module = "dev.daymor.ultima-nexus.jvm:annotation-jvm", version = "latest.release" }
ultima-nexus-jvm-processor = { module = "dev.daymor.ultima-nexus.jvm:processor-java-spring-hibernate", version = "latest.release" }
----

If these catalog entries are present, they take precedence over the fallback coordinates.
