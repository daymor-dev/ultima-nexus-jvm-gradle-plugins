= Native Image Plugins
:description: GraalVM Native Image plugin for Ultima Nexus JVM Gradle Plugins
:toc: left
:toclevels: 3

The `plugins-native` module provides GraalVM Native Image compilation support.

== GraalVM Native Plugin

Applies the `org.graalvm.buildtools.native` plugin and configures default native image build settings.

Plugin ID:: `dev.daymor.ultimanexus.jvm.gradle.feature.graalvm-native`

[source,kotlin]
----
plugins {
    alias(libs.plugins.ultimanexus.jvm.feature.graalvm.native)
}
----

=== What It Does

* Applies the GraalVM Native Build Tools plugin
* Configures `nativeCompile` task for building native executables
* Configures `nativeTest` task for running tests in native mode

=== Extension Configuration

[source,kotlin]
----
graalvmNativeConfig {
    verbose.set(false)
    quickBuild.set(false)
    fallback.set(false)
    richOutput.set(true)
}
----

=== Properties Configuration

[source,properties]
----
graalvmNative.verbose=false
graalvmNative.quickBuild=false
graalvmNative.fallback=false
graalvmNative.richOutput=true
----

=== Building and Testing

[source,bash]
----
# Build a native executable
./gradlew nativeCompile

# Run tests in native mode
./gradlew nativeTest
----

=== Prerequisites

* GraalVM JDK 22.3+ (includes `native-image` by default)

== Properties Reference

[cols="2,1,3"]
|===
| Property | Default | Description

| `graalvmNative.verbose`
| `false`
| Enable verbose GraalVM Native Image output

| `graalvmNative.quickBuild`
| `false`
| Enable quick build mode (faster but less optimized)

| `graalvmNative.fallback`
| `false`
| Allow fallback to JVM when native image cannot be built

| `graalvmNative.richOutput`
| `true`
| Enable rich output during native image compilation
|===

== When to Use Native Compilation

Native compilation is beneficial when:

* *Startup time matters* -- Native executables start in milliseconds instead of seconds
* *Memory footprint matters* -- Native images use significantly less memory
* *Deploying to serverless/containers* -- Faster cold starts and smaller footprint

Native compilation is less suitable when:

* *Development velocity is priority* -- Native compilation is slow (3-10 minutes)
* *Dynamic features are required* -- Reflection and dynamic proxies need explicit configuration

TIP: For Spring Boot native applications, use the bundles in xref:spring-boot-plugins.adoc[Spring Boot Plugins] which combine this plugin with Spring Boot AOT processing.
